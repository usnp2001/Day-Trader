========================================================================
             當沖看盤選股交易平台 - SQLite 資料庫結構與欄位資訊 (Phase 4)
========================================================================

本系統採用 SQLite 作為本機資料儲存庫（檔案名稱為 trading_platform.db）。
資料庫內共包含 5 張資料表，全面支援多使用者模擬帳戶（Option B 隔離）、JWT 身份認證、
角色分工（管理者與一般使用者）、可用資金動態調整以及 FinMind 與 yfinance 混合大數據評價選股。

以下為各張資料表的詳細用途說明與 Schema 欄位規格：

------------------------------------------------------------------------
1. 使用者憑證表 (users)
------------------------------------------------------------------------
● 用途說明：
  儲存註冊使用者的帳號、密碼雜湊與權限角色。
  - 角色分為 'admin'（管理者）與 'user'（一般使用者）。
  - 密碼儲存前會結合動態 UUID 進行加鹽（salt）並以 SHA-256 進行不可逆雜湊防護。

● 欄位資訊：
  ┌─────────────────┬─────────┬─────────────┬─────────────────────────────────┐
  │ 欄位名稱        │  型態   │   主鍵/條件 │ 欄位說明                        │
  ├─────────────────┼─────────┼─────────────┼─────────────────────────────────┤
  │ username        │ TEXT    │ PRIMARY KEY │ 使用者帳號（唯一不重複）        │
  │ hashed_password │ TEXT    │ NOT NULL    │ 加鹽後的 SHA-256 密碼雜湊值     │
  │ salt            │ TEXT    │ NOT NULL    │ 隨機生成之鹽值 (UUID hex)       │
  │ role            │ TEXT    │ NOT NULL    │ 帳戶角色（admin / user）        │
  └─────────────────┴─────────┴─────────────┴─────────────────────────────────┘

------------------------------------------------------------------------
2. 帳戶資產表 (account)
------------------------------------------------------------------------
● 用途說明：
  儲存每位使用者的模擬交易可用現金餘額。
  - 當新用戶註冊或管理者開立帳戶時，自動在此表為該用戶初始劃撥 NT$ 10,000,000 可用資金。
  - 使用者可透過介面呼叫 API 對本表的 cash 數值進行自由變更。

● 欄位資訊：
  ┌──────────┬─────────┬─────────────┬─────────────────────────────────┐
  │ 欄位名稱 │  型態   │   主鍵/條件 │ 欄位說明                        │
  ├──────────┼─────────┼─────────────┼─────────────────────────────────┤
  │ username │ TEXT    │ PRIMARY KEY │ 關聯之使用者帳號 (users.username)│
  │ cash     │ REAL    │ NOT NULL    │ 該使用者之可用現金餘額          │
  └──────────┴─────────┴─────────────┴─────────────────────────────────┘

------------------------------------------------------------------------
3. 持有庫存部位表 (positions)
------------------------------------------------------------------------
● 用途說明：
  儲存各個使用者目前持有的未平倉持股明細。
  - 採用複合主鍵 (username, symbol) 確保各個使用者的庫存部位完全隔離。
  - 當買進/賣出交易完成後，系統會自動重新計算該使用者此股票的平均成本（buy_price）。
  - 若使用者持股數量（qty）歸零，該股票之紀錄會自動從此表中刪除。

● 欄位資訊：
  ┌───────────┬─────────┬─────────────┬────────────────────────────────┐
  │ 欄位名稱  │  型態   │   主鍵/條件 │ 欄位說明                       │
  ├───────────┼─────────┼─────────────┼────────────────────────────────┤
  │ username  │ TEXT    │ PRIMARY KEY │ 關聯之使用者帳號 (users.username)│
  │ symbol    │ TEXT    │ PRIMARY KEY │ 股票代碼（如 2330.TW, AAPL）   │
  │ name      │ TEXT    │ NOT NULL    │ 股票名稱（如 台積電, Apple）   │
  │ qty       │ INTEGER │ NOT NULL    │ 持股股數（正數為多單，負數為空單)│
  │ buy_price │ REAL    │ NOT NULL    │ 該使用者之平均持股成本價格     │
  └───────────┴─────────┴─────────────┴────────────────────────────────┘

------------------------------------------------------------------------
4. 成交歷史紀錄表 (orders)
------------------------------------------------------------------------
● 用途說明：
  儲存各個使用者的所有下單委託交易歷史紀錄（包含已成交 FILLED、委託中 PENDING、已取消 CANCELLED）。

● 欄位資訊：
  ┌──────────────┬─────────┬─────────────┬─────────────────────────────┐
  │ 欄位名稱     │  型態   │   主鍵/條件 │ 欄位說明                    │
  ├──────────────┼─────────┼─────────────┼─────────────────────────────┤
  │ order_id     │ TEXT    │ PRIMARY KEY │ 委託單唯一代碼（如 MOCK-1001）│
  │ username     │ TEXT    │ NOT NULL    │ 關聯之使用者帳號 (users.username)│
  │ symbol       │ TEXT    │ NOT NULL    │ 股票代碼（如 2317.TW）      │
  │ action       │ TEXT    │ NOT NULL    │ 交易方向（BUY / SELL）      │
  │ price        │ REAL    │ NOT NULL    │ 委託價格（市價單填入 0.0）  │
  │ qty          │ INTEGER │ NOT NULL    │ 委託股數                    │
  │ order_type   │ TEXT    │ NOT NULL    │ 委託種類（LIMIT / MARKET）  │
  │ status       │ TEXT    │ NOT NULL    │ 委託狀態（FILLED / PENDING） │
  │ exec_price   │ REAL    │ NOT NULL    │ 實際成交價格                │
  │ timestamp    │ TEXT    │ NOT NULL    │ 成交時間字串（格式為 HH:MM:SS)│
  └──────────────┬─────────┬─────────────┬─────────────────────────────┘

------------------------------------------------------------------------
5. 股票搜尋與篩選快取表 (stock_metadata)
------------------------------------------------------------------------
● 用途說明：
  此表為全系統的快取快顯中心，在多使用者下為共享資源（不隔離）。
  首頁的多條件選股篩選器與看板的 Autocomplete 模糊搜尋功能，皆直接查詢此快取表。
  本階段結合 FinMind 籌碼大數據與 yfinance 多因子評價指標，大幅擴展了評價與基本面維度。

● 欄位資訊：
  ┌────────────────┬─────────┬─────────────┬────────────────────────────────┐
  │ 欄位名稱       │  型態   │   主鍵/條件 │ 欄位說明                       │
  ├────────────────┼─────────┼─────────────┼────────────────────────────────┤
  │ symbol         │ TEXT    │ PRIMARY KEY │ 股票代碼（如 2330.TW, 5347.TWO）│
  │ name           │ TEXT    │ NOT NULL    │ 公司簡稱（如 台積電, Intel）    │
  │ price          │ REAL    │ NOT NULL    │ 當前收盤價/最新報價            │
  │ change         │ REAL    │ NOT NULL    │ 今日漲跌價差                   │
  │ change_percent │ REAL    │ NOT NULL    │ 今日漲跌幅 (%)                 │
  │ volume         │ INTEGER │ NOT NULL    │ 最新單日成交量（股數）         │
  │ pe_ratio       │ REAL    │ DEFAULT NULL│ 本益比（可能為 NULL）          │
  │ ma5            │ REAL    │ DEFAULT NULL│ 5日移動平均價（可為 NULL）      │
  │ ma20           │ REAL    │ DEFAULT NULL│ 20日移動平均價（可為 NULL）     │
  │ stockId        │ INTEGER │ DEFAULT 0   │ 股票代號數值（台股如 2330，美股 0）│
  │ pb_ratio       │ REAL    │ DEFAULT NULL│ 股價淨值比 PBR                 │
  │ dividend_yield │ REAL    │ DEFAULT NULL│ 現金股利殖利率 (%)             │
  │ foreign_net_buy│ INTEGER │ DEFAULT 0   │ 外資單日買賣超（股數）         │
  │ trust_net_buy  │ INTEGER │ DEFAULT 0   │ 投信單日買賣超（股數）         │
  │ dealer_net_buy │ INTEGER │ DEFAULT 0   │ 自營商單日買賣超（股數）       │
  │ margin_balance │ INTEGER │ DEFAULT 0   │ 融資當日餘額（張數）           │
  │ short_balance  │ INTEGER │ DEFAULT 0   │ 融券當日餘額（張數）           │
  │ revenue_yoy    │ REAL    │ DEFAULT NULL│ 月營收年增率 YoY (%)           │
  │ roe            │ REAL    │ DEFAULT NULL│ 股東權益報酬率 ROE (%)          │
  │ revenue_growth │ REAL    │ DEFAULT NULL│ 營收年增率 (%)                 │
  └────────────────┬─────────┬─────────────┬────────────────────────────────┘
